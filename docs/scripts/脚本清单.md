### 1.1 migrate_v0_to_v1.py - Schema 迁移

**功能**: 将 LanceDB 中的 Schema 元数据从 v0 版本升级到 v1 增强版本。

**使用方式**:
```bash
python scripts/migrate_v0_to_v1.py --config=conf/agent.yml [选项]
```

**核心参数**:
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--config` | 配置文件路径 | 必填 |
| `--namespace` | 命名空间 | 自动选择 |
| `--extract-statistics` | 收集统计信息 | false |
| `--extract-relationships` | 提取关系 | true |
| `--import-schemas` | 迁移后导入 Schema | false |
| `--clear` | 导入前清除数据 | false |
| `--force` | 强制重新迁移 | false |
| `--llm-enum-extraction` | LLM 增强枚举提取 | false |

**完整迁移示例**:
```bash
python scripts/migrate_v0_to_v1.py \
  --config=conf/agent.yml \
  --namespace=test \
  --extract-statistics=true \
  --extract-relationships=true \
  --import-schemas \
  --llm-enum-extraction \
  --clear \
  --force
```

### 1.2 diagnose_schema_ddl.py - DDL 抽样诊断

**功能**: 抽样检查 LanceDB 中存储的 DDL 是否存在缺逗号/截断等问题。

**使用方式**:
```bash
python scripts/diagnose_schema_ddl.py --config=conf/agent.yml --namespace=<name>
```

**核心参数**:
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--config` | 配置文件路径 | conf/agent.yml |
| `--namespace` | 命名空间名称 | 自动选择 |
| `--catalog` | Catalog 过滤 | 空 |
| `--database` | Database 过滤 | 空 |
| `--limit` | 抽样表数量 | 5 |
| `--random` | 随机抽样 | false |
| `--compare-db` | 拉取数据库 DDL 对比 | false |

**示例**:
```bash
# 随机抽样 10 张表并对比数据库 DDL
python scripts/diagnose_schema_ddl.py \
  --config=conf/agent.yml \
  --namespace=test \
  --limit=10 \
  --random \
  --compare-db \
  --output=/tmp/ddl_sample.json
```

### 1.9 generate_business_config.py - 业务术语配置生成器

**功能**: 从数据架构设计文档和指标清单 Excel/CSV 生成业务术语配置文件 `business_terms.yml`，支持多行表头、LLM 增强和 DDL 注释合并。

**核心特性**:
- 支持 Excel (.xlsx) 和 CSV (.csv) 两种输入格式
- 智能处理多行表头（数据架构3行表头、指标清单2行表头）
- 自动检测工作表（优先 "Sheet1"，否则第一个）
- 支持 LLM 增强的术语消歧和冲突解决
- 可合并现有 DDL 注释进行增强
- 生成三种映射关系：`term_to_table`、`term_to_schema`、`table_keywords`

**使用方式**:
```bash
python scripts/generate_business_config.py --config=conf/agent.yml --namespace=<name> [选项]
```

**核心参数**:
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--config` | 配置文件路径 | 必填 |
| `--namespace` | 目标命名空间 | 必填 |
| `--arch-xlsx` | 数据架构详细设计 Excel 路径 | - |
| `--arch-csv` | 数据架构详细设计 CSV 路径（兼容） | - |
| `--arch-sheet-name` | 数据架构工作表名称/索引 | 自动检测 |
| `--metrics-xlsx` | 指标清单 Excel 路径 | - |
| `--metrics-csv` | 指标清单 CSV 路径（兼容） | - |
| `--metrics-sheet-name` | 指标清单工作表名称/索引 | 自动检测 |
| `--output` | 输出 YAML 文件路径 | `conf/business_terms.yml` |
| `--merge-ddl` | 合并现有 DDL 注释 | false |
| `--use-llm` | 启用 LLM 增强 | false |
| `--include-confidence` | 输出置信度分数（调试用） | false |
| `--min-term-length` | 最小术语长度 | 2 |
| `-v, --verbose` | 详细输出（-v INFO, -vv DEBUG） | - |

**输入文件格式要求**:

数据架构详细设计 Excel（3行表头）:
- 第1行: 分组标题（资产目录/业务属性/技术属性/管理属性）
- 第2行: 实际列名（物理表名、字段名、分析对象等）⭐
- 第3行: 列说明/注释
- 关键列: `物理表名`、`字段名`、`属性（中文）`、`属性业务定义`、`分析对象（中文）`

指标清单 Excel（2行表头）:
- 第1行: 实际列名（指标名称、来源dws模型等）⭐
- 第2行: 子分类/详细说明
- 关键列: `指标名称`、`来源dws模型`、`业务定义及说明`

**推荐用法示例**:

```bash
# 1. 基础用法：从Excel生成（自动检测工作表）
python scripts/generate_business_config.py \
  --config=conf/agent.yml \
  --namespace=test \
  --arch-xlsx=/path/to/数据架构详细设计v2.3.xlsx \
  --metrics-xlsx=/path/to/打铁指标清单v2.4.xlsx \
  --verbose

# 2. 合并 DDL 注释（推荐）
python scripts/generate_business_config.py \
  --config=conf/agent.yml \
  --namespace=test \
  --arch-xlsx=/path/to/数据架构详细设计v2.3.xlsx \
  --metrics-xlsx=/path/to/打铁指标清单v2.4.xlsx \
  --merge-ddl \
  -v

# 3. 指定工作表名称
python scripts/generate_business_config.py \
  --config=conf/agent.yml \
  --namespace=test \
  --arch-xlsx=/path/to/数据架构详细设计v2.3.xlsx \
  --arch-sheet-name="模型设计" \
  --metrics-xlsx=/path/to/指标清单v2.4.xlsx \
  --metrics-sheet-name=0

# 4. LLM 增强模式（术语消歧）
python scripts/generate_business_config.py \
  --config=conf/agent.yml \
  --namespace=test \
  --arch-xlsx=/path/to/数据架构详细设计v2.3.xlsx \
  --use-llm \
  --include-confidence \
  --output=conf/business_terms_debug.yml

# 5. 仅处理数据架构（不处理指标清单）
python scripts/generate_business_config.py \
  --config=conf/agent.yml \
  --namespace=test \
  --arch-xlsx=/path/to/数据架构详细设计v2.3.xlsx \
  --merge-ddl
```

**输出文件示例** (`conf/business_terms.yml`):
```yaml
# Business Terms Configuration
# Generated by generate_business_config.py
# Namespace: test
# Total mappings: 150 table terms, 320 schema terms

term_to_table:
  有效线索: ["dws_clue_valid", "dwd_clue_base"]
  试驾预约: ["dws_test_drive", "dwd_test_drive_base"]
  首触渠道: ["dws_channel_first_touch"]
  
term_to_schema:
  客户姓名: ["customer_name", "dwd_customer.customer_name"]
  手机号码: ["mobile_phone", "dwd_clue_base.mobile_phone"]
  
table_keywords:
  首触: "dws_channel_first_touch"
  有效线索: "dws_clue_valid"
  铂智3X: "dim_vehicle_model"
```

**配置加载优先级**:
生成后的 `business_terms.yml` 按以下优先级加载:
1. 环境变量 `BUSINESS_TERM_CONFIG` (文件路径)
2. `conf/business_terms.yml` (项目本地)
3. `~/.datus/conf/business_terms.yml` (用户全局)

**故障排除**:

| 问题 | 解决方案 |
|------|----------|
| "0/5559 有效行" | 检查 `--arch-sheet-name` 是否正确，确认Excel表头行数 |
| 列名匹配失败 | 使用 `-vv` 查看实际列名，检查Excel格式 |
| 找不到工作表 | 使用 `pd.ExcelFile(xlsx).sheet_names` 查看可用工作表 |
| 内存不足 | 分批处理或使用 CSV 格式 |

## 二、模型下载

### 2.1 download_models_cn.sh - 国内模型下载

**功能**: 下载 HuggingFace 模型，支持国内镜像和 ModelScope 回退。

**使用方式**:
```bash
bash scripts/download_models_cn.sh
```

**特点**:
- 自动检测并切换 HuggingFace 镜像
- 支持 ModelScope 作为备选源
- 下载到 `~/.datus/models/`

**下载内容**:
1. `qdrant/all-MiniLM-L6-v2-onnx` - 核心嵌入模型（必需）
2. `BAAI/bge-reranker-large` - 重排序模型（可选）

**手动下载示例**:
```bash
# 仅下载嵌入模型
bash scripts/download_models_cn.sh
# 出现提示时按 N 跳过重排序模型
```

---

### 2.2 download_models_docker.sh - Docker 模型下载

**功能**: Docker 构建专用模型下载，下载到 `docker/huggingface/` 目录。

**使用方式**:
```bash
bash scripts/download_models_docker.sh
```

**用途**: 在构建 Docker 镜像前预下载模型，避免网络超时。

**下载内容**:
1. `qdrant/all-MiniLM-L6-v2-onnx` → `docker/huggingface/fastembed/`
2. `BAAI/bge-reranker-large` → `docker/huggingface/reranker/`

**Docker 构建流程**:
```bash
# 1. 下载模型
bash scripts/download_models_docker.sh

# 2. 构建镜像
docker build -t datus-agent:latest .

# 3. 运行容器
docker run -p 8080:8080 \
  -v /path/to/agent.yml:/root/.datus/conf/agent.yml \
  datus-agent:latest
```

## 五、故障排除

### 5.1 常见问题

| 问题 | 解决方案 |
|------|----------|
| LLM 回退失败 | 禁用 LLM 回退：`--llm-fallback=false` |
| 模型下载超时 | 使用国内镜像：`bash scripts/download_models_cn.sh` |
| 权限错误 | 确保数据库用户有访问系统表权限 |
| 存储路径冲突 | 检查 `agent.yml` 中的 namespace 配置 |

### 5.2 回滚操作

```bash
# 从备份恢复
bash scripts/backup_storage.sh  # 先创建新备份
rm -rf ~/.datus/data/datus_db
mv ~/.datus/data/datus_db.backup_* ~/.datus/data/datus_db
```
